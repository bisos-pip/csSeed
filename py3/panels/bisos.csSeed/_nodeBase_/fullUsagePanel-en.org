* +
* cs.G.seedOfThisPlant  and  b.cs.G.plantOfThisSeed
* Seeding and planting are the concepts of extending and customizing an exisiting CS.
* The .cs to be extended and/or customized  is called the "seed"
* The .pcs (planted cs) which expands the seed is called "planted"
* The .spcs (spread planted cs) is assmed to be executing from a specific directory (e.g. lcntProc.spcs)
* The .npcs (niched planted cs) applies to customizations that are in the context of realms (site, platform)
* _IMPORTANT_ :: It is possible to implement .pcs as an independent layer on top of .cs without interdependencies
* _IMPORTANT_ :: plantedCsu is both a feature of csxu-seed.cs, wuth a small aspects in bisos.b
* _IMPORTANT_ :: scope of plantyedCsu in  bisos.b needs to be documented NOTYET <2026-02-06 Fri>
* csSeed/py3/bisos/csSeed/cmnds_seedInfo.py is available as a generic machinary cmndsSeedInfo.kwSeedInfo
* The Planted CS file .pcs is executed twice. Once as itself and once as plantedCsu. So far this has been harmless.
