- csCmndEval does not assume all.

- It Supports doContinue
- It supports pyPipe=True in CsCmnd.pars; when set, opResults of that command become --pyStdin of the next

** Implementation Notes
- pyPipe=True in a CsCmnd's pars is eval-internal: it is stripped from the subprocess invocation line.
- The receiving command reads pyStdin as a normal CS parameter (--pyStdin="<value>").
- On doContinue=True after failure, pyStdin is cleared (not forwarded).
- csCmndEval appears in the examples menu alongside csCmndsRun.

** Needs Testing
- Basic case: csCmndEval all -- runs all selected csCmnds in sequence, same as csCmndsRun.
- doContinue=False (default): a failing command halts the sequence.
- doContinue=True: a failing command is skipped, pyStdin cleared, sequence continues.
- pyPipe=True: opResults of command N are injected as --pyStdin into command N+1's invocation.
- pyPipe absent or False: pyStdin is not forwarded (each command is independent).
- Chain of multiple piped commands: N pipes into N+1 pipes into N+2.
- pyPipe=True on the last command in the list: opResults surfaced as csCmndEval's own opResults.
- Verb-filtered invocation (not "all"): only matching verbs run, pipe chain respects filtered order.
